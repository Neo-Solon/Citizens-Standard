## STEP 6 — Browser verifier (new)
`methodology.html` now runs 9 of the 12 replication packages **live in the page** (Pyodide / CPython
compiled to WebAssembly). It needs two things to be pushed alongside it:

- [ ] `replication/web_manifest.json` — tells the page which scripts to run, in what order, and which
      files to fetch. Regenerate with `python replication/build_web_manifest.py` after touching any package.
- [ ] `.nojekyll` (repo root) — GitHub Pages runs Jekyll by default; this guarantees every file under
      `replication/` is served verbatim.
- [ ] `.github/workflows/replication.yml` — runs the FULL suite (all 12) on push + weekly, fails the
      build if any package stops reproducing, uploads the HTML report.

All **11** paper packages run in the browser. Two are flagged heavy (Empirical Validation ~25 MB,
Distribution ~30 MB): the page warns, asks before downloading, and excludes them from "Run all 9".

The manifest fetch now uses `cache: 'no-store'` — without it the browser serves a stale package list
after every deploy (this is what made the reordering look like it hadn't taken effect).

**It will NOT work from `file://`** — browsers block a local page from reading local files. The page
now detects this and says so, with the fix (`python -m http.server`). On the live site it runs normally.
