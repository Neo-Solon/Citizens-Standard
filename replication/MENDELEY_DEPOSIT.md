# Note on the deferred Mendeley Data deposit

## Status

The SSRN page for Paper 2 — *The Citizens Standard as Counterfactual Benchmark* — currently has no associated Mendeley Data deposit. This is intentional and temporary.

## Why deferred

`replication.py` in this folder is the Python implementation of the empirical method described in Paper 2. It uses the same data sources, the same K1/K2 formulas, the same Dec-Dec CPI deflation chain, and the same Fisher-equation real-return compounding as the paper. The methodology family is the same.

What it does not currently do is reproduce the paper's published cell values to the dollar. Paper 2's Tables 1, 2, and 3 were generated by an interactive computation built during the paper's drafting (in a separate workspace) before this script was ported to Python. The two implementations agree on direction and broad magnitude — Mode B reliably exceeds median actual retirement outcomes, the equity-vs-principal decomposition shows compound returns dominate, Cohort D depends sensitively on forward equity returns — but produce different absolute numbers.

A Mendeley Data deposit is meant to be the citable, DOI-stamped, frozen replication record for a paper. Depositing a script that does not reproduce the paper's headline numbers would undermine the deposit's purpose more than not depositing anything. The right move is to fix the alignment first, then deposit a clean version.

## What "fixing the alignment" requires

The reconciliation work needs to do three things:

1. Identify the precise points of divergence between the paper's interactive computation and this script. The most likely candidates are: the source of the real GDP growth input (chain-weighted BEA series vs. nominal GDP minus CPI, floored at zero), the post-2025 projection convention for years that compound at the central scenario rate, and the M2 series treatment in years where the prior-year value is missing or anomalous.

2. Pick whichever approach is methodologically defensible and document the choice. If the script's approach is correct, update the paper's tables to match. If the paper's approach is correct, update the script. Either way, the answer should be a single set of numbers that both the published paper and the deposited script produce.

3. Re-run the four cohorts plus the Cohort D scenarios plus the two stress tests, verify the outputs match the paper's tables to the dollar (or update the paper to match the script), and only then deposit on Mendeley.

## When this happens

This is planned work for a focused future session. The Mendeley field on the SSRN submission page accepts deposits at any point — at submission or after. Until the reconciliation is done, the SSRN page will show no associated dataset.

## What readers can do in the meantime

- Read `replication.py` directly to see the methodology in code form
- Run it with `python3 replication.py` to see what the Python implementation produces
- Compare those outputs against the paper's tables — the script is internally consistent and the differences from the paper are bounded (cohort-by-cohort within roughly 25–35 percent of the paper's central-scenario figures)
- Open an issue on the parent repository if anything in the script looks structurally wrong

## What this file is not

This is not an erratum on the paper. The paper's central empirical claims — Mode B reliably exceeds median actual outcomes; the wealth advantage derives overwhelmingly from compound returns; the framework is most accurately characterized as a structural retirement-security architecture rather than a monetary-stimulus mechanism — survive any reasonable reconciliation between the paper and the script. The pending work concerns the precision of cell values, not the validity of findings.
